Spring核心容器

我们知道，我们使用spring的话会写我们的xml文件，里面定义各种各样的bean。定义好了bean之后，使用spring核心容器加载我们的xml文件，然后通过核心容器获取并使用我们定义好的bean。

一、容器相关

  • BeanFactory是IOC容器的顶层接口，初始化BeanFactory对象时，加载的bean延迟加载(类似懒汉单例模式)

  • ApplicationContext接口是Spring容器的核心接口，初始化对象时bean立即加载（饿汉单例模式）

  • ApplicationContext接口提供基础的bean操作相关方法，通过其他接口扩展其功能

  • ApplicationContext接口常用初始化类

    • ClassPathXmlApplicationContext （建议使用）
    • FileSystemXmlApplicationContext

二、bean相关

  • bean基本参数
参数 参数描述
id bean 的唯一标识符
name bean的别名 可以起多个用空格分隔
class bean类型，静态工厂类，FactoryBean类
scope 设置bean的实例数量 singleton表示单例 prototype表示非单例 默认 单例
init-method 生命周期相关 初始化方法
destroy-method 生命周期相关 销毁前方法
autowire 自动装配 byTape 根据类型自动装配 或者byName 根据名称自动装配
factory-method bean工厂方法 应用于静态工厂或者实例工厂
factory-bean 实例工厂bean
lazy-init 参数为true 懒汉单例模式或者false 饿汉单例模式
